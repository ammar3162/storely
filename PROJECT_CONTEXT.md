## قواعد معمارية مهمة

1. لا يوجد middleware.ts — المنطق بـ src/proxy.ts فقط.
2. كل استعلام منتجات يفلتر بـ branch_id من sessionStorage — هذا مقصود.
3. أي إضافة منتج لازم تحدد branch_id صراحة.
4. التحقق من "صنف موجود" يستخدم .limit(1) بدل .maybeSingle().
5. صفحة الأدمن تستخدم service role key عبر API endpoint، ليس client عادي.
6. أسماء متغيرات البيئة الصحيحة: NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY.

## سجل تاريخي — أخطاء حرجة تم إصلاحها

1. RLS كان معطّل على profiles و branches — تم تفعيله وإصلاح recursion
2. صفحة الأدمن كانت تظهر مستخدم واحد فقط — تم إصلاحه بـ service role key
3. باغ تكرار المنتجات بالمشتريات (الأخطر) — فلتر branch_id خاطئ + .maybeSingle() يفشل صامتاً مع التكرار
4. منتجات بـ branch_id فاضي عبر 3 مؤسسات — تم الإصلاح والتنظيف
5. متغيرات بيئة فاضية على Vercel — تم اكتشافها وإصلاحها
6. تسريب مفتاح service role بالخطأ — تم توليد مفتاح جديد فوراً
